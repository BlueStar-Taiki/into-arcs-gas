# 星空撮影イベント申し込み GAS 運用マニュアル

## 対象

- スプレッドシート: `星空撮影イベント申し込み`
- スプレッドシート ID: `1f7s_LQfI6GrVy5O99pTOrbk2XE1r3m4p5b5okIA0M-o`
- フォーム: `https://forms.gle/yw1MrgfEw6JrzVrp7`
- GAS ソース: `gas/star-photo-application/src`

## 導入前の確認

1. Google フォームの回答先が対象スプレッドシートになっていることを確認する。
2. 回答タブ名が `フォームの回答 1` であることを確認する。
3. 回答タブのヘッダーがスキーマ記載の5項目と一致することを確認する。
4. このGASを対象スプレッドシートのコンテナバインド型プロジェクトとして配置する。

`.clasp.json`、Webhook URL、カレンダーID、認証情報、フォーム回答などの個人情報はリポジトリへコミットしない。

## Script Properties

GASエディタの「プロジェクトの設定」→「スクリプト プロパティ」で次を設定する。

| プロパティ | 必須 | 値 |
|---|---|---|
| `DISCORD_WEBHOOK_URL` | Discord通知を使う場合は必須 | 対象チャンネルのWebhook URL |

Webhook URLそのものを `設定` シートやソースコードへ記載しない。

## 初回セットアップ

1. GASエディタへ `src` 配下の8ファイルを追加する。
2. プロジェクトのタイムゾーンも `Asia/Tokyo` に設定する。
3. `setupApplicationFormSheet()` をGASエディタから初回に手動実行し、権限を承認する。
4. `申込管理`、`設定`、`ログ` のヘッダー、固定行、列幅、プルダウン、初期設定値を確認する。
5. `設定` シートの `EVENT_NAME`、`CONTACT_NAME`、必要なら `REPLY_TO_EMAIL` を編集する。
6. GASエディタの「トリガー」からトリガーを追加する。
   - 実行する関数: `onFormSubmit`
   - イベントのソース: スプレッドシートから
   - イベントの種類: フォーム送信時
7. スプレッドシートを再読み込みし、`INTO-ARCS管理` メニューが表示されることを確認する。

`onFormSubmit` は権限が必要なメール送信と外部HTTP通信を行うため、単純トリガーではなくインストール型トリガーが必要である。GASエディタから `onFormSubmit` を直接実行してはならない。

## 動作確認

1. テスト用のメールアドレスでGoogleフォームから1件送信する。
2. `フォームの回答 1` に元回答が追加されていることを確認する。
3. `申込管理` に同内容と申込IDが追加されていることを確認する。
4. 初期状態が `新規受付`、`未登録` であることを確認する。
5. 確認メールを受信し、メール送信状況が `送信済み` になることを確認する。
6. Discordへ通知され、Discord通知状況が `通知済み` になることを確認する。
7. `ログ` にフォーム処理完了ログがあることを確認する。
8. `申込管理` の対象データ行を1行選択し、管理メニューからメール再送とDiscord再通知をそれぞれ確認する。
9. テスト申込に個人情報を使った場合は、確認後に運用ルールに従って削除する。

## エラー時

- メールまたはDiscordだけが失敗しても、申込管理への転記は維持される。
- 対応する通知状況が `送信エラー` / `通知エラー` になり、`ログ` に詳細が残る。
- 原因を修正後、申込管理の対象行を1行選択し、管理メニューから再送・再通知する。
- フォーム項目名を変更した場合は `config.js` のヘッダー定義も同時に更新する。

## 注意点

- `setupApplicationFormSheet()` は既存の業務データを削除しない。初期設定値も既存キーを上書きしない。
- 管理シートに想定外のヘッダーがある場合は、安全のためセットアップを中断する。
- プルダウンと日時書式は初期状態で1000データ行分設定する。必要に応じて `VALIDATION_ROW_COUNT` を増やす。
- 同時送信時の行競合を避けるため、フォーム処理はスクリプトロックを使用する。
- 現時点ではカレンダー連携を実装していない。関連列は将来拡張用である。
- Discord通知には申込者名と参加人数が含まれる。通知先チャンネルの閲覧権限を限定する。
