# 星空撮影イベント申し込み データ設計

## 基本方針

- 運用タイムゾーンは `Asia/Tokyo` とする。
- `フォームの回答 1` は Google フォームが書き込む生データとして扱い、GAS から編集しない。
- 業務上のステータスや通知結果は `申込管理` に保持する。
- 列は位置ではなくヘッダー名で解決する。
- Discord Webhook URL はシートに保存せず、Script Properties から取得する。

## フォームの回答 1

| ヘッダー | 用途 |
|---|---|
| タイムスタンプ | フォーム受付日時 |
| お名前 | 申込者名 |
| 参加人数 | 参加者数 |
| 緊急連絡先電話番号 | 当日の緊急連絡先 |
| メールアドレス | 確認メール送信先 |

このシートは `setupApplicationFormSheet()` の作成対象ではない。フォーム連携によって存在すること、および上記ヘッダーが揃っていることだけを検証する。

## 申込管理

| ヘッダー | 初期値・用途 |
|---|---|
| 申込ID | `STAR-yyyyMMdd-HHmmss-XXXXXXXX` |
| 受付日時 | フォームのタイムスタンプ |
| お名前 | フォーム回答 |
| 参加人数 | フォーム回答 |
| 緊急連絡先電話番号 | フォーム回答 |
| メールアドレス | フォーム回答 |
| ステータス | `新規受付` |
| メール送信状況 | `未送信` → `送信済み` / `送信エラー` |
| Discord通知状況 | `未通知` → `通知済み` / `通知エラー` |
| カレンダー登録状況 | `未登録`（将来連携用） |
| イベントID | 将来のカレンダー連携用 |
| 内部メモ | 運用担当者用 |
| 最終更新日時 | 作成・通知状態更新時刻 |

プルダウン値:

- ステータス: `新規受付`, `確認中`, `予約確定`, `キャンセル`, `対応不要`
- メール送信状況: `未送信`, `送信済み`, `送信エラー`
- Discord通知状況: `未通知`, `通知済み`, `通知エラー`
- カレンダー登録状況: `未登録`, `登録済み`, `登録エラー`, `削除済み`

## 設定

| キー | 初期値 | 用途 |
|---|---|---|
| EVENT_NAME | 星空撮影イベント | メール・通知に表示するイベント名 |
| CONTACT_NAME | INTO-ARCS | メール送信者表示名・署名 |
| REPLY_TO_EMAIL | 空欄 | メール返信先。空欄なら指定しない |
| DISCORD_MENTION | 空欄 | Discord本文の任意メンション |

`DISCORD_MENTION` は Discord の意図しないメンションを防ぐため、Webhook payload の `allowed_mentions.parse` を空にしている。実際に通知を飛ばすメンションが必要な場合は、運用方針に合わせてコード側の許可対象も明示的に変更する。

## ログ

| ヘッダー | 用途 |

| レベル | `INFO`, `WARN`, `ERROR` |
| 処理名 | 発生元の関数・処理 |
| 申込ID | 関連する申込ID |
| メッセージ | 要約 |
| 詳細 | スタックトレースなど |

## 個人情報

緊急連絡先とメールアドレスは申込管理に必要な範囲で転記する。Discord通知には緊急連絡先とメールアドレスを含めない。スプレッドシート、GASプロジェクト、Discordチャンネルの共有範囲は最小限にする。
